# Git Tasks Internship Repository

## Overview

This repository contains a series of tasks completed as part of an internship focused on learning Git and version control practices. Each task demonstrates a specific concept, ranging from basic commands to advanced workflows.

---

## Repository Structure

The repository is organized into task-based folders:

```
git-tasks/
│
├── task-1/
├── task-2/
├── task-3/
├── task-4/
├── task-5/
├── task-6/
├── task-7/
├── task-8/
├── task-9/
└── task-10/
```

Each folder contains:

* Source files used for the task
* A README explaining the implementation
* Output screenshots (where applicable)

---

## Tasks Summary

### Task 1: Repository Initialization and Basic Workflow

Initialized a repository, created files, committed changes, and worked with branches.

### Task 2: .gitignore and File Tracking

Configured `.gitignore` and verified tracked vs ignored files.

### Task 3: Undoing Changes

Used `git restore`, `git reset`, and `git revert` to undo changes at different stages.

### Task 4: Merge Conflicts

Simulated and resolved merge conflicts between branches.

### Task 5: Interactive Rebase

Cleaned commit history using interactive rebase and squash operations.

### Task 6: Git Stash

Temporarily saved and restored uncommitted changes using stash.

### Task 7: Cherry-Picking

Applied specific commits from one branch to another.

### Task 8: Git Hooks

Implemented a pre-commit hook to enforce checks before committing.

### Task 9: Remote Collaboration

Worked with remote repositories, feature branches, and pull requests.

### Task 10: Advanced Workflow and Recovery

Simulated force pushes, commit loss, and recovery using `git reflog`.

---

## Key Concepts Covered

* Version control fundamentals
* Branching and merging
* Conflict resolution
* History rewriting (rebase)
* Safe undo operations
* Collaboration using GitHub
* Automation using Git hooks
* Recovery of lost commits

---

## Learning Outcome

This repository demonstrates a structured approach to understanding Git through practical implementation. It highlights both basic and advanced workflows used in real-world development environments.

---

## Conclusion

The tasks collectively provide a strong foundation in Git and version control, covering everything from simple commits to complex scenarios involving history rewriting and recovery.
